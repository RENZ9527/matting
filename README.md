# Matting LoRA Training

This repository only documents the environment setup and LoRA training command for the matting task.

## Environment

```bash
conda create -n matting python=3.12
conda activate matting
pip install -r requirements.txt
```

Download FLUX.1-Kontext-dev and place it locally:

```bash
hf download black-forest-labs/FLUX.1-Kontext-dev \
  --local-dir /path/to/FLUX.1-Kontext-dev
```

Expected model directory:

```text
/path/to/FLUX.1-Kontext-dev/
├── tokenizer/
├── tokenizer_2/
├── text_encoder/
├── text_encoder_2/
├── ae.safetensors
└── flux1-kontext-dev.safetensors
```

## Dataset

Download the matting dataset from Hugging Face:

```bash
hf download Renz-7/new_matting \
  --repo-type dataset \
  --local-dir ./datasets/new_matting
```

After downloading, the dataset should include original images, alpha masks, trimaps, and metadata files.

Expected directory structure:

```text
merged_matting_dataset/
├── train/
│   ├── mask/
│   ├── original/
│   ├── trimap/
│   ├── filenames_train.txt
│   └── metadata.txt
├── train.json
└── train.txt
```

Before training, edit the paths in `create_txt.py`, then generate a new `filenames_train.txt`:

```bash
python create_txt.py
```

The generated `filenames_train.txt` contains only two columns:

```text
original mask
```

Example:

```text
/path/to/merged_matting_dataset/train/original/sample_0001.jpg /path/to/merged_matting_dataset/train/mask/sample_0001.png
```

Trimaps are generated by the training code from the alpha mask, so they do not need to be listed in `filenames_train.txt`.

## Paths to Modify

Before training, update the local paths in the following files:

```text
create_txt.py
├── IMAGE_DIR      # training original images
├── GT_DIR         # training alpha masks
└── OUTPUT_FILE    # generated filenames_train.txt

scripts/Kontext_matting3_lora.sh
├── --dataset_base_path      # training dataset root
├── --dataset_metadata_path  # filenames_train.txt
├── --model_paths            # FLUX.1-Kontext-dev path
├── --output_path            # checkpoints, logs, and evaluation outputs
├── --eval_file_list         # validation file list
└── --resume                 # resumes from the latest checkpoint in output_path

configs/accelerate_config.yaml
└── gpu_ids                  # GPU id used for training
```

The validation path is also hard-coded in the training and evaluation code. Update these if your validation dataset is not stored at the original AIM-500 path:

```text
scripts/train.py
└── base_dir for matting evaluation

models/utils.py
├── base_dir for matting evaluation
└── eval_args.gt_path for matting metrics

utils/eval_matting.py
└── dataset_paths["aim"] validation split file
```

## LoRA Training

Before training, modify the dataset path, model path, output path, GPU id, and other training parameters directly in:

```text
scripts/Kontext_matting3_lora.sh
```

Then launch training:

```bash
bash scripts/Kontext_matting3_lora.sh
```
